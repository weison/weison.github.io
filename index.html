发表于 2017-02-19 |

版本兼容过程

flex布局

  • 旧版本dispaly: box;
  • 过渡版本dispaly: flexbox;
  • 以及现在的标准版本display: flex;

Android

  • 2.3 开始就支持旧版本 display:-webkit-box;
  • 4.4 开始支持标准版本 display: flex;

IOS

  • 6.1 开始支持旧版本 display:-webkit-box;
  • 7.1 开始支持标准版本 display: flex;
  • 设为Flex布局以后，子元素的float、clear和vertical-align属性将失效。

基本概念

容器及项目

采用flex布局的元素，称为Flex容器,简称”容器”，它所有子元素自动成为容器成员，称为Flex项目（flex item）,简称”项目”

主轴main axis及交叉轴cros axis

  • 容器中默认存在两个轴，水平的轴（main axis）与交叉轴(cross axis)。
  • 主轴开始的位置（与边框的交叉点）叫做main start,结束的位置叫做main end
  • 主轴开始的位置（与边框的交叉点）叫做cross start,结束的位置叫做cross end
  • 项目默认沿主轴排列。单个项目占据主轴的空间叫main size。占据交叉轴的空间叫cross size

容器的属性

flex-direction

决定主轴的方向

  • row 默认值，主轴为水平方向，起点在左端
  • row-reverse 主轴为水平方向，起点在右端
  • column 主轴为垂直方向，起点在顶端

flex-wrap

项目都排在一条线（又称”轴线”）上。flex-wrap属性定义，如果一条轴线排不下，如何换行

  • nowrap默认值，代表不换先
  • wrap 换行，第一行在上方
  • wrap-reverse 换行，第一行在下方

flex-flow

flex-direction属性和flex-wrap属性

justify-content

主轴上的对齐方式

align-items

交叉轴上的对齐方式

align-content

定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用

项目的属性

order

项目的排列顺序,数值越小，排列越前

flex-grow

项目放大比例，默认为0，即存在剩余空间也不放大

flex-shrink

项目缩小比例，默认为1，即空间不足时缩小项目

flex-basis

在分配多余空间之前，项目占据的主轴空间

flex

是flex-grow,flex-shrink,flex-basis的缩写，默认为0 1 auto。后两个属性可选

align-self

单独定义项目的交叉轴对齐方式

字符串操作

发表于 2017-02-11 |

javascript中的对象是一个泛指，所有的东西都是对象，类是指把对象具体细分Number、String、Function、Object、Date、RegExp，实例是类中的一个具体东西，如{},[],’123’;

length获取长度(空格也会计算)

charAt根据索引值获取对应字符

charCodeAt获取字符对应的unicode编码

每一个字母、汉字、特殊字符，都有它对应的unicode码值

String.formCharCode 根据unicode编码，返回对应字符

indexOf从左到右查到字串第一次出现的索引值

str.indexOf(searchValue[, fromIndex])

  • searchValue 一个字符串表示被查找的值。
  • fromIndex 表示调用该方法的字符串中开始查找的位置。可以是任意整数。默认值为 0。如果 fromIndex < 0 则查找整个字符串（如同传进了 0）。如果 fromIndex >= str.length，则该方法返回 -1，除非被查找的字符串是一个空字符串，此时返回 str.length。
  • 区分大小写的
  • 假如没有查找到内容，会返回-1
  • fromIndex < 0 则查找整个字符串（如同传进了 0）
    String.prototype.indexOf()

search()搜索字符串，并返回子字符的索引值

lastIndexOf() 从右往左查找符合要求的字符索引值

字符串对比（相当于字符unicode编码对比）

str.substr(start[, length])截取

返回字符串中从指定位置开始到指定长度的子字符串。

  • start
    开始提取字符的位置。如果为负值，则被看作 strLength + start，其中 strLength 为字符串的长度（例如，如果 start 为 -3，则被看作 strLength-3）。
  • length
    可选。提取的字符数。
    假如start为负数，且abs(start)的长度大于str.length,则截取会从0开始

String.prototype.substr()

str.substring(indexStart[, indexEnd])

返回字符串两个索引之间（或到字符串末尾）的子串。
String.prototype.substring()

  • 如果任一参数小于 0 或为 NaN，则被当作 0。

slice截取

str.slice(beginSlice[, endSlice])提取字符的一部分，并返回新的字符串
String.prototype.slice()

  • slice 不修改原字符串，只会返回一个包含了原字符串中部分字符的新字符串
  • slice() 提取新字符串不包括 endSlice。

跟substring有两点不同：

  1. 不同根据两个数字的不同而调换位置

  2. 当索引值是负数时，计算方式为str.length + 索引

通过截取来隐藏部分内容并添加省略号

toUpperCase和toLowerCase 英文大小写转换

  1. toUpperCase() 小写英文转换成大写
  2. toLowerCase() 大写英文转换成小写

replace(旧字符，新字符)

  • 在不使用正则的情况下，每次执行，只会替换一个

split 将字符折分成数组

str.split([separator][, limit])通过把字符串分割成子字符串来把一个 String 对象分割成一个字符串数组。
String.prototype.split()

  • 不写任何参数，表示不进行拆分，直接将整个字符串转换为数组；
  • 填写””即将每个字符将分成一个数组；
  • 假如带参数，则将参数当划分线，划分后，被当作分割符的内容不保留

join 将数组转换成一个字符串

利用git进行版本控制

发表于 2017-02-11 |

git作用

  • 可以回溯到之前的任意版本
  • 解决多人协作时的冲突问题

github

  • geek平台，可以找到开源项目以及作远程仓库之用
  • 可视化工具github desktop

svn与git的区别

  • svn采用的是集中式的管理方式,所有代码都存放在中
  • git采用的是分布式的管理方式

使用git

  • 复制仓库 git clone [url]
  • 设置贡献者,用于在多人协作时识别coder身份
  1. git config –global user.name “名字”
  2. git config –global user.email “邮箱地址”
  3. 查看开发者名称 git config –global user.name
  4. 查看开发者邮箱 git config –global user.email
  5. 查看所有git的项目配置文件 git config –list
  • 分支
  1. 为了保证新增功能不影响主项目的开发，而创建一个与主分支一样的副本文件，当这个副本开发相对完善后再合并到主分支，形成一个新的稳定版本
  2. 默认分支 master
  • git的三个区
  1. 工作区，正在编写代码的开发环境
  2. 暂存区
    • 作为工作区与版本库的桥梁，减少误操作的风险
    • 当工作区的文件在误操作时可以通过暂存区复原，也保证未编写完整的代码提示到版本库上
    • 分支处理，当要临时切换到另外的分支进行开发工作时，可以把正在编写的文件提交到暂存区，然后再切换到另一个分支，当有需要时再切换回来，从暂存区取出代码
  3. 版本库
  • git 命令
  1. git status 查看工作区和暂存区的状态，是否有经过编辑
    • Changes not staged for commit 说明已跟踪文件的内容发生了变化，但还没有放到暂存区
  2. git add 文件名 => git add index.html 将工作区文件添加到暂存区
  3. git add . 将工作区所有文件添加到暂存区
  4. git commit -m “修改的内容” => git commit -m “change on file” 将文件从暂存区添加到版本库
  5. git commit -a -m “修改内容” => git commit -a -m “hello” 将文件同步保存到暂存区并添加到版本库,-a就是简写的add
  6. git log 查看提示记录,当提交记录过多时，不会一下全部打印出来，不过可以按回车来查看，然后按“q”键即可退出
  7. git diff 对比命令
  8. git diff 直接使用对比工作区与暂存区的内容
  9. git diff –cached 或 git diff –staged 暂存区与版本库内容对比
  10. git diff master 对比本地与版本库
    1.撤消
  11. git reset HEAD 文件名 => git reset HEAD drag.js 将你出去到暂存区的drag.js文件取回到暂存区，相当于取消提交
  12. git checkout – 文件名 => git checkout – drag.js 将文件回复到版本库上的内容
  13. git commit –amend => git commit -a -m “修正上一版本错误” –amend 覆盖上一次提交到版本库的文件，再重新上传一次
    1.删除
  14. git rm 文件名 => git rm test.txt ，当工作区的文件删除后，使用git rm test.txt 删除暂存区的文件
  15. git rm -f 文件名 => git rm -f test.txt 同时删除本地及暂存区的文件
  16. git rm –cached => git rm -cached test.txt 只删除暂存区的文件
    1.恢复
  17. git checkout commitId 文件名 => git checkout 50e7b8e0610 drag.js 使用git log 查看提交的id,然后恢复具体的文件
  18. git reset –hard commitID => git reset –hard 50e7b8e061 恢复到某个特定版本
    • git reset –hard head^ 返回上一个版本
    • git reset –hard head~3 返回前3个版本
  19. git reflog 查看所有操作，当切换到旧版本后git log没有了新的记录，可以通过git reflog查看所有操作，然后然后git reset –hard commit 来回来新版本
    1.提交代码
  20. git remote 远端名称
  21. git remote -v 远程地址
  22. git push 名字 分支名
  23. haha

合并分支 git merge [分支名称]
删除分支 git branch -d [分支名称]

创建仓库

  1. touch readme.md创建描述文件
  2. git init 初始化本地目录
  3. git add [fileName]让本地文件提交到暂存区
  4. git commit -am '[更新描述]' -am的意思，-a代表修改的文件,m代表删除的文件,所以这句话的意思是把所有添加、修改、删除都提交
  5. git remote add [远程地址名称] [远程地址路径]如git remote add origin abc.git就相当于把abc.git起了个名字叫origin,下次提交的时候写git push origin master就相当于推送内容到远端origin下的master分支
  6. git pull更新同步一下远端的代码，

分支操作

  1. git branch -a查看当前本地的所有分支，红色的不加星号的就是远程分支
  2. git branch dev创建一个dev分支
  3. git checkout dev切换到dev分支
  4. git push -u origin dev把本地的分支推送到远程

代码合并

  1. git add .把最新的修改添加到更新列
  2. git commit -am [更新描述]
  3. git pull [远端地址] [分支名称]
  4. git push [远端地址] [分支名称]把内容更新到远端
  5. git merge [分支名称]
  6. git merge master把主干master上的所有代码合并到当前分支
  7. 假如冲突(CONFLCT)，要先解决冲突

代码回溯

  1. git log -l 3查看最近3次的提交
  2. 找到SHA码
  3. git reset --hard [SHA码]将本地的代码强制回到了[SHA]对应的文件(但复原后直接拒绝，会被拒绝),可以使用
    git reset -f --hard [SHA码]来强行提交，不过这样会把在这个版本后的记录都会清除（不建议使用）,为了避免
    这种情况，可以使用git revert [SHA码]来返回指定的版本，github会把这次回滚理解成一次新的修改，这样会保留记录安全了很多。
