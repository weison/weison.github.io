自动化构建工具

发表于 2016-03-18 |

fis

FIS3 , 为你定制的前端工程构建工具

启动服务

  1. 安装 npm install fis -g
  2. 发布 fis release
  3. 启动服务器 fis server start
  4. 打开项目投送路径 fis server open

gulp

简介

基于node.js环境的管道(pipe)流程式的前端自动构建工具
通常流程是：建立task->读取(src)文件储存在内存中->经过各种处理（pipe压缩、合并、改名等）->dest(输出)

运行环境

安装nodejs

  1. 安装node.js -> node.js官网
  2. node -v检测是否安装成功
  3. npm install gulp -g 全局安装gulp
  4. npm install gulp --save-dev 局部安装gulp
  5. gulp -v检测是否成功

安装gulp并初始化环境

  1. npm init初始化项目环境,并进行项目的相关配置，注意package.json中的name不要命令为gulp否则会被拒绝安装
  2. npm install --global gulp全局安装gulp
  3. npm install --save-dev gulp开发环境中安装gulp
  4. 创建gulpfle.js来对gulp进行开发和配置

gulp命令

var 依赖 = require('依赖包名')

gulp.src()数据源,表示需要处理的文件

  1. js/a.js 指定确定的文件名
  2. js/*.js 目录下的所有js文件
  3. js/**/*.js 目录及其子目录下所有的js文件
  4. !js/a.js 除了js下a.js以外的所有文件
  5. [js/a.js,js/b.js] 还可以一个数组
  6. *.+(js css) 匹配项目根目录下，所有后缀名为js或css的文件。

pipe

gulp任务可以是同步或者异步的。在异步任务确定任务完成，可以通过调用函数的done。

文件重命名

合并文件

  • gulp-rev 给每个文件算也一个hash码，然后把这些文件的名字修改掉
  • gulp-rev-replace 根据修改后的hash文件，修改匹配路径
  • gulp-useref 通过注释的方式让gulp实现如css,js合并的操作
  • gulp-filter 从文件集合中筛选出特定的文件，处理完成后通过restore放后文件流中继续下一步操作
  • gulp-uglify 压缩js代码
  • gulp-csso 压缩css
