angularjs1x

发表于 2016-03-20 |

应用场景

前端视图与后台数据结合得比较紧密

  1. CRUD（增加Create、读取Read、更新Update、删除Delete）应用程序
  2. 单页应用SPAS(single Page Applications)，利用路由进行整站式的开发

MVC设计模式

  1. model 数据模型，后台数据
  2. viwe 视图，浏览器呈现的页面
  3. controller M和V的连接器，负责逻辑运算

MVVM数据双向绑定

数值改变影响视图，视图改变数据

  1. 数据改变视图

  2. 视图改变数据

表达式

将数据内容绑定到html里，与ng-bind类似，不过表达式如果在未完整渲染时会显示这条表达式，而ng-bind会在渲染完再显示

作用域

从内到外查找数据

$scope

挂载数据

局部作用域

$scope.$apply动态更新视图

$rootScope

全局作用域

依赖注入

依赖angular来注入服务生成对象，不需要考虑需要传递什么参数，而是通过行参来搞定

好处

  1. 解耦
  2. 自动创建对象

问题

  1. 压缩时，编译器无法分辨出这是依赖注入的行参可能把$scope压缩成dd

directive指令

定义：扩展html属性，带有前缀ng-
作用：

ng-app 初始化,会在自身以及其内部元素起到初始化作用

这样有利于在进行开发的时间 ，某些部分并不需要使用angular开发

ng-model 数据绑定，元素值绑定到html中

  1. ng-model-option 配置ng-model
  2. updateOn 控制model更新的时机

ng-repeat 重复html元素

  1. 通过in方法来指定遍历内容

ng-controller 连接数据和视图

ng-cloak 在表达式解释完前内容的display为none,解释完成后display为block

ng-bind-template可以串连多个表达式

ng-non-bindable 不对表达式进行解释，直接显示出来

只会影响标签及它里面的内容

ng-value

ng-bind

ng-class

ng-class=”{样式名:true,样式名2:true}”
可以通过动态的控制布尔值来启用或者取消样式

ng-style行内样式

可以写在行内，也可以抽取出来

ng-href=""

ng-attr-具体的属性名(suffix)

通用的属性写法,主要是考虑到ng没有添加的指令，可以通过这种形式添加

Dom操作

  1. ng-show

  2. ng-hide
    通过css的display属性来控制显示隐藏

  3. ng-if
    直接通过DOM节点的增加和删除来操作

  1. ng-switch有选择性在显示或隐藏元素
  • ng-switch-default 当on为true时显示的内容
  • ng-switch-when="string" 当on的内容匹配string时显示
  1. ng-open
    针对html5的detail标签

ng-init

利用ng-repeat实现数组循环嵌套，利用ng-init进行初始化操作

ng-include="'路径'" 通过模板的方式引入文件

标签指令

a 阻止了系统默认事件

例如空链接点击会刷新

ng-options

类似于ng-repeat,ng-options=”具体数据的哪一个属性 for 其中一条数据 in 数据源”,
需要配合ng-model一起使用

novalidate

阻止form表单默认的行为和样式

服务service

注入依赖引进来
1.都是以$开始的

$http 完成ajax操作

angular.module('模块化名称'.[])模块化思想

作用

  1. 防冲突
  2. 建立依赖关系

过滤器

filter : '要查找的内空'[: true] 找出符合条件的内容

  1. 它针对的值都是value值
  2. 假如加上true后，则要求完全匹配,默认情况下 filter: ‘r’ 是可以查找到
    值为red和blue的数目，当如果写成 filter : ‘r’ : true后，则只能找到完全匹配的值，单个r

currency 修改数字格式

limito 截取部分内容

可以截取字符串，数据，甚至是数组的内容

orderBy : '排序的依据' [: true] 排序

假如加上true，则从大到小排

number 把字符转换成不带单位，但带分隔符的数字

默认四舍五入保留3位小数，可以通过number后面的数字来控制保留的位数

uppercase/lowercase转换为大写/小写

data 将时间的毫秒数转换成时间

过滤器是可以组合使用的

只要在表达式内在管道符之间加上管道符即可

在js中使用过滤器

  1. 注入$filter服务
  2. 在$scope里执行绑定的数据

自定义

自定义过滤器

  1. 通过filter方法
  2. 将过滤器定义在模块下面
  3. 可以像普通过滤器一样，通过管道符来调用
  4. 也可以通过js的$scope.属性名 = $filter(‘过滤器名称’)(‘过滤的内容’)来过滤

自定义服务

factory

provider

两者的区别在于当服务共用时，provider（供应商）可以通过config来根据不同的需求对服务进行重新配置

config

  1. 固定组合，服务名字+Provider，比如服务名叫hello，那个在进行配置时，配置的名字就叫helloProvider
  2. 供应商一定要写在config中

模块下的通信

通过angular.module(‘模块名’,[‘依赖包的名字’])来通信

工具方法

angular.isArray() 判断内容是否为数组

angular.isDate()

判断内容是否为时间对象

angular.isDefined 判断内容是否已存在

如果存在，就会返回假

angular.undefined

判断内容是否不存在

angular.isFunction

判断内容是否为函数

angular.isObject()

判断内容是否为对象

angular.isNumber()

判断内容是否为数字

angular.idString()

判断内容是否为字符

angular.isElement()

判断内容是不是一个元素

angular.version

判断当前所使用的angularJs的版本

angular.equals()

判断两个元素是否相等

angular.foEach(操作的内容,回调函数,结果对象)

angular.fromJson/toJson

json与字符串转换
类似于原生的JSON.parset() 和 JSON.stringify()

  1. 把字符串转换成json

  2. 把json格式化成str,第二个参数可以让字符串格式更容易看懂

angular.lowercase/uppercase 字母大小写转换

angular.element()

angular中包含了一些jq的方法，具体看api
当同时引进jq后，angular.element ==== $

angular.bootstrap 动态初始化app,而且可以支持同时初始化多个模块

angular.inject注册器

$scope.$watch 监听数据变化

$scope.$apply

监听funciton里面的内容是否有变化，一旦发生变化，就更新视图内容

run()

变量全局操作，初始化全局的一些参数，可以不用通过连接controller来进行
操作，它只能挂载在rootScope下

插件

表单验证

注意：

  1. 通过name的方式来查找

$valid 有效的

当验证成功时，会返回true

$invalid 无效的

当验证失败时，会返回true

$pristine 原始值，初始值

原始值没经过修改，会返回true

$dirty

脏值，已经修改过的值后，这个值会返回true

$error

返回所有验证信息，只要这个值为true，就证明失败了
假如为false，则代表验证通过
而且要考虑优先问题，如果一个内容，同时需要是否为空，是否符合正则，是否符合最小位数
那个当内容未填写为空时，为显示为{“require”:true,”pattner”:false,”minLength”:false}

required 验证是否为空

如果为空，返回true

minLength 最小字符长度

如果达到条件，则返回false

ng-pattern 使用正则验证

主要的css样式

  1. ng-valid{} 验证通过
  2. ng-invaild{} 验证不通过
  3. ng-prisine{} 原始数据样式
  4. ng-dirty{} 脏处理
