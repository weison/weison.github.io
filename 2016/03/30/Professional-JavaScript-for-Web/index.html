DOM (Document Object Model)文档对象模型

它把HTML的每个组成部分看成某个类型的节点,这些节点又包含不同类型的数据。借助DOM提供的API（Application Programming Interface）可以增加(createElement)、删除（removeChild）、替换(replaceChild)或修改任何节点。

BOM (Browser Object Model)

人们习惯上也把所有针对浏览器的JavaScript 扩展算作BOM的一部分

  1. 弹出新浏览器窗口的功能window.open()
  2. 移动、缩放和关闭(window.close())浏览器窗口的功能
  3. 像XMLHttpRequest 和IE 的ActiveXObject 这样的自定义对象。

在HTML 中使用JavaScript

script元素

  1. 默认情况下，浏览器会根据script的先后顺序来加载
  2. async和defer都只能应用于外部引用的js
  3. async加载和渲染后续文档元素的过程将和 script.js 的加载与执行并行进行（异步）
  4. defer 同时加载后续内容及js文件，但执行要在所有元素元素解释完成后,遇到</html>标签后执行，DOMContentLoaded事件触发前执行。
  5. 外部javascript应带有.js后缀。但这个后缀不是必须的，因为浏览器不会检测包含Javascript的文件扩展名。这样一来，使用jsp,php或者其它服务器端口语言动态生成的js代码也就成为可能。但必须确保服务器能返回正确的MIME类型.

延迟脚本defer

1.考虑默认情况下，js需要下载并执行完后再加载下面的内容(阻塞下载)，如果将script放在头部，那么在js下载的时候，页面会一片空白（页面会在遇到body时才开始渲染页面）。在现实当中，延迟脚本并不一定会按照顺序执行，也不一定会在DOMContentLoaded 事件触发前执行，因此最好只包含一个延迟脚本。

  1. 不保证按照指定它们的先后顺序执行，因为要确保两个js互不依赖
  2. 指定async 属性的目的是不让页面等待两个脚本下载和执行，从而异步加载页面其他内容。
    为此，建议异步脚本不要在加载期间修改DOM

嵌入代码与外部文件

虽然html没有硬性规定不可以在html中包含Js,不过考虑到以下优点，还是建议把js放在一个独立的文件中

  1. 可维护性，如果将js遍及到不同页面，这样维护起来很麻烦。
  2. 可缓存，浏览器能够根据具体的设置缓存链接的所有外部JavaScript 文件。当两个页面使用同一份文件时，那么浏览器会只下载一次。加快网页加载速度

基本概念

语法

区分大小写

text 和 Text 是不同的变量

标识符

标识符是指变量，函数、属性或者函数的参数名称。规则如何

  1. 首字符以字母，_或$开头；
  2. 其他字符可以是字母、下划线、$或数字
  3. 按照惯例，命名方式应该采用驼峰式命名(newFun,myCar)

注释

单行

以两个斜杠开头

多行

以一个斜杠和一个星号（/）开头，以一个星号和一个斜杠（/）,第2，第3行只是为了添加可读性而添加

严格模式"user strict"

它是一个编译指示，用于告诉浏览器切换到严格模式.一些不确定的行为将得到处理，某些不安全的操作将会抛错

  1. 它可以在整个Js文件内使用 “user strict”
  2. 也可以单独在函数内使用

语句

javascript以”;”代表一个语句的结束。为了

  1. 避免压缩时出错
  2. 不用系统在解释时需要推测是否要加“;”，提高性能
  3. 开发人员可放心地删除多余的空格

变量

变量属于松散类型的数据，可以用来保存任何类型的数据。它不过是一个用于保存值的点位符而已。

使用var 声明的变量，只会存在于它的作用域下

  1. 如果把变量声明在函数内，那么这个变量将会在退出函数时(执行完)时失效
  • 不过为了避免全局污染，造成代码难以维护还是不建议使用全局变量的定义
  • 给未声明的变量赋值，严格模式下会报错ReferenceError
