页面布局

发表于 2016-04-02 |

所谓布局就是浏览器以正确的大小在内容放在正确的位置上。

影响布局的属性

display呈现方式

  1. 设置元素的显示方式
  2. display:block|inline-block|inline|none
属性 特征 默认元素
block 可设宽高，默认宽度为父级元素的宽度 独占一行 div,p,h1-h6
inline 设置宽高无效，宽度为内部内容的宽度，文本流形式 a,span,img
inline-block 可设宽高，默认为内部内容的宽度，文本流形式,注意假如inline-block内部内容的宽度加上这行内容的宽度超过整行的宽度时，inline-block元素会独占一行 input,textarea,select,button
none 设置元素不可显示

display:none与visibility:hidden的区别

  1. display为none时，相当于脱离了文本流，后面的元素会占据它原来的位置
  2. visibility：hidden，它只是隐藏了内容，但还是会占据着它原来的位置

    position定位

  3. 通过top,right,bottom,left设置元素边缘距离参照物的距离，z-index设置层级
  4. 当同时设置2个相对的值时而且未设置宽高值的情况,如top和bottom值时
    |relative|会根据先后定位顺序来覆盖|
    |absolute或fixed|会改变元素的尺寸直到符合规则|
    而如果设置了宽高，则按定义的顺序来，如先定义left,后定义right,则使用right

    relative 相对定位

    相对于其自身的原来位置的距离

absolute 绝对定位

相对于其最近定位了positon:absolute|fixed的元素的距离

fixed 固定定位

相对于浏览器来进行定位

sticky 粘性布局

内存泄露

发表于 2016-04-01 |

由于程序设计的缺陷，让我们不需要的内容不受控制的驻留在内存当中，造成了资源的浪费。js的内存泄露很大一部分由于引用清理了，内存没清除掉

内存模型分为两部分，堆和栈，堆用于存放复杂类型的数据，栈用于存放基本类型的数据

检查工具

chrome timeline

chrome timeline cg

chrome profile Take Heap Snapshot

如何避免内存泄露

避免使用eval，它可以把字符当成函数来调用，但它的作用域全都是挂在window的全局下面的

谨慎使用全局全量，尤其是全局变量存在DOM引用

绑定事件的时候，要注意当DOM被移除时解绑事件

注意闭包和循环引用时容易出现问题

从输入URL到网页呈现，发生了什么

发表于 2016-03-22 |

用户派浏览器去特定的地址（url）去取货（请求资源），浏览器收到地址后利用自己的知识找到它的门牌号码(ip地址)，然后帮用户打包好信息，通过各种手段（get,post,trace…）等方法跟主机进行3次握手(问：是否可以连接？答：可以。浏览器说那就发送了)。主人收到消息后，通过分析信息（cookie,请求的资源地址…）给信息打包给浏览器，通过获取本地文件是否存在（cache-control,expirse）或浏览器上的标识（etag,last-Modified)来让浏览器直接使用它原有的资源，浏览器接受到资源后，从上到下开始下载资源，并且是html和css并行下载与渲染，并遇到js时，为了不用重复搬砖（重新渲染DOM），所以它会让js下载完成后继续下载，然后再渲染页面。
