正则表达式含义

表达一个计算机所能理解的规则，是强大的字符串匹配工具

写法

js风格 new RegExp(‘a’,’i’)

perl风格 /a/i

加上iignore，忽略英文大小写

gglobal，找出全部内部

|或

[]元字符（方括号）

[n1n2]x 代表或，即只要符合括号内的任意一个，都算是符合要求

[n1-n2]范围,[0-9],[a-z]

[^n1-n2]排除,[^a-z]排除a-z,[^a-z0-9]还可以连着用，代表同时排队a-z和0-9

方法

search() 查找符合规则的字符,返回索引值

match() 把符合规则的内容提取出来

replace(规则，替换的内容)替换符合规则的内容

re.test(str)检验字符串是否符合规则

*注意，test方法在验证的时候，只要有一部分符合要求，它都会返回true

所有为了避免这种情况可以用

^字符串开头，$结尾来让规则达到完整匹配

转义

\d代表任意数字

.点，代表任意字符，尽量不要用，容易出问题

\w代表数字、字符串、,类似于[a-z0-9]

\s space,代表空格，tab等不要打印、不可显示的字符

\D 代表除了0到9，等价于[^0-9]

\W 代表除了a-z,0-9,下划线以外的字符，等价于[^a-z0-9]

\S 代表非空格的字符

量词,个数

{n}正好有几个

{n,m}最小n次，最多m次

{n,}从n到无限大

? 可有可无,()代表分组

()? 代表括号里面的内容可有可无

+量词，若干(多少都可以)

获取字符串中的数字
1.使用charAt()获取字符，然后利用隐性转换对比字符是不是>=0且<9

常用正则表达式

  1. 正整数 /^[1-9]\d*$/
  2. 整数 /^[+]?\d+(.)?(\d+)?$/
  3. 中文 /^[+]?\d+(.)?(\d+)?$/

闭包closure

发表于 2016-02-23 |

定义

闭包

  1. 函数嵌套函数,内部的函数可以引用外部函数的变量和参数，参数和变量不会被垃圾回收机制收回。
  2. 这是一种数据结构,而且必须满足两个条件
  • 必须是返回一个函数
  • 这个返回的函数里必须引用到外部栈区的内容

垃圾回收机制

好处

希望一个变量长期存在于内存中

不污染全局变量(隔离作用域)

避免因为名字一样，而造成在js合并过程中，后面的变量覆盖前面的变量

私有成员的存在

作用

保存变量环境，在循环中直接找到对应元素的索引

封装，模块法代码

从而做到信息隐藏，或分可见和不可见的内容

注意事项

ie下引发内存泄露

堆和栈

会在函数执行的时候分配，执行完了清理。

栈（用来存放原始类型undefinder,null,Number,String,Boolean）

  1. 由编译器自动分配释放，存放函数的参数和变量值。他们通常会在被调用的分配，调用完即释放

堆（存放引用类型,Object）

  1. 一般由程序员控制分配释放，若程序不释放，程序结束时可由OS回收。它的生命周期由虚拟机的垃圾回收算法来决定（并且一旦成为孤儿对象就能被回收）。当程序运行完了，垃圾清理机制会检查堆里面的对象是否还在被栈引用，如果没有，则清空

发表于 2016-02-12 |

作用域概念

在特定的空间或范围内对数据进行读写操作

  1. 作用：读、写操作
  2. 域：空间、范围、区域…

一对script标签就是一个域

  • 有域的地方就会发生预解释和逐行解释代码的过程

  • 在解释的时候，是自上而下来解读的，一个处理完再处理另外一个

  • 自上而下的解释过程中，上一个全局script的变量与函数在下一个全局script中仍然可以被读取写入

一个函数是一个局部作用域

作用域链

函数内部也算一个作用域，当在预解释过程中，在函数里面没有找到a的变量值时，会从子作用域(fn1)跳到父级作用域查找，这种从里到外的查找的过程叫作用域链

一个参数就相当于一个局部变量

js解释过程

1.预解释 ‘找一些东西’var function 参数

  1. 把声明的变量找出来并默认值设为undefined
    a=undefined
  2. 所有的函数声明，整个提取出来
    fn1 = function fn1(){alert(1);
  3. 遇到重名的情况
  • 当变量与函数重名,函数覆盖变量（因为变量值是undefined，
    而函数是整个函数块，所以会采用函数）
  • 函数与函数重名时，后面的的覆盖前面的

2.逐行解读代码

表达式 + - * / % Number() 参数，所有能改变变量值的操作都叫表达式，它是可以改变预解释的值的。

函数外部读取函数内值的方法

把全局变量放进函数内，然后把想读取的值赋予这个全局变量

局部函数调用(把全局函数放到函数内调用)

if``for``do while都不是作用域

数据类型的真假

数据类型有：

  1. 数字(包含NaN )
  2. 字符串
  3. 布尔值
  4. 函数
  5. 对象(element,[],{},null)
  6. undefined

真

  1. 所有非0的数字(1,-1,1231,1.231)
  2. 所有的非空字符串(‘afbd’，’不可思异’,’ ‘有空格)
  3. true
  4. 所有的函数(function(){})
  5. 能找到的元素
  6. [],{}数据与json就像一个仓库，无论内面有没有放东西，
    这个仓库都真实存在，所以永远返回true

假

  1. 0、NaN
  2. 空字符串(‘’无空格)可以理解为空字符串就像在放屁一样，
    但里面包含了空格，就是想说但没说，所以会返回true
  3. false
  4. 不能找到的元素
  5. null
  6. undefined 未定义

性能优化的方向

加载性能

一般最核心的就是减少请求数，因为每次请次数据都会产生3次TCP握手，如果每个资源都发出请求，那会很浪费时间。当合并完后要注意代码的压缩，减少体积

首屏加载时间 一般是指Dom ready时间

  • initial connection 网络等待时间
  • Waiting(TTFB)向服务器请求数据，服务器返回第一个数据包的时间，反应服务器性能
CDN

网络分发服务，主要是请求一些静态的资源.从最靠近的资源节点中获取数据

避免link中的空href和img中的空src

因为它也会发送请求的

使用Gzip压缩
把css放在顶部

减少重绘重排的几率

把js放在底部

避免js的阻塞

避免使用css表达式

主要是针对ie下的expression

把css和js抽取成外部文件

这样可以充分利用缓存机制，但并不代表内联就不好，内联的好处就是减少网络请求

get请求比post请求的速度要快
  • get请求常用于获取数据
  • post常用于修改数据
减少cookie的大小

因为在每次请求的时候都会带上cookie，如果cookie过大，会造成资源浪费

html prefetch

实现域名的预解释和页面的预加载
HTML5 prefetch-赖小赖小赖

$.getscript 动态加载script
对于非首屏的图片都使用lazyload

避免资源的浪费，同时避免了资源下载的抢占资源，比如说有100张图片，如果同时加载，那么通过有限的并发时，这100张图片就会占用网络带宽

js方面的

尽量不要使用with和eval

性能低下，而且有作用域转换问题，容易引起bug

优化DOM操作

高性能JavaScript DOM编程以及重排与重绘

渲染速度

用户交互体验速度

动画流畅性
