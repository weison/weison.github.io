promise是commonJs的一种规范，它是JS中异步操作方式,Jquery的deferred就是使用了该规范，而在新版本的ecmaScript 6中已经加入了原生的promise。
promise允许异步是可以按一步顺序出现的

ecmaScript

状态

pending(等待)->resolve(成功)->reject(失败)
成功的状态会调用成功对应的函数，失败的状态会调失败的函数

then

它是属于promise构造函数原型链上的一个方法Promise.prototype.then=funciton(){},所以如果希望在外部调用的话，只要在函数执行完成后return这个promise对象即可
调用resolve和reject状态所对应的回调函数

利用promise利用链式操作，实现任务队列式执行

catch

用于接受失败的回调，或者充当then的第二个回调函数(reject)

all

多个异步操作的函数都返回正确之后再执行回调函数

race只要有其中的回调状态是成功的，那就会执行race里的方法

when(函数1()，函数2()).then()对应all方法

基本数据类型与引用数据类型的区别

基本数据类型操作的是值，数据类型操作的是内存地址引用

  • 基本数据类型是值的传递，值的修改不 会影响到其它变量

  • 引用数据类型是值的引用（可相当把内存地址标识赋给了它），值修改会影响引用这个地址的所有变量

  1. 定义一个变量
  2. 开辟一个新的空间，然后把属性名和属性值保存在这个空间中，并且有一个空间地址
  3. 把空间地址给了这个变量，变量并没有储存这些数据，而且储存了这个空间引用地址
  4. 如果过程中把这个地址也给了这个变量，另一个变量也是存储这个地址，此时两个变量操作的是同一个空间地址
  5. 其中一个改变了，另外一个也跟着改变

形参

  • 可以理解成是一个变量，它的作用是，在我们定义函数时，无法知道参数的实际内容是什么，因此采用形参的方式来代表需要处理的数据内容，当执行这个方法时，只要把实际数据代表形参即可
  • 定义一个形参，假如没有传值，那个这个形参的默认值为undefined

多态

一个功能的多种形态，通过参数的不同实现不同的功能

arguments是函数内置的接收参数的机制

  • 系统自带的，不管是否传递过参数，也不管有没有定义的形参，arguments一直存在
  • 这是一个类数组的形式，索引值从0开始代表第几个传递的参数
  • arguments.length代表传递参数的个数

arguments.callee 代表当前函数本身

return

  • 如果想获取函数内部的值，可能采用return的方法来把结果返回出去。
  • 如果函数没有写return内容，或者直接写return,那么系统默认返回undefined
  • 在函数体内，return后面的语句不会被执行，利用这个特征，可以控制函数体在哪个中止执行
