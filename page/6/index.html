作用域概念

在特定的空间或范围内对数据进行读写操作

  1. 作用：读、写操作
  2. 域：空间、范围、区域…

一对script标签就是一个域

  • 有域的地方就会发生预解释和逐行解释代码的过程

  • 在解释的时候，是自上而下来解读的，一个处理完再处理另外一个

  • 自上而下的解释过程中，上一个全局script的变量与函数在下一个全局script中仍然可以被读取写入

一个函数是一个局部作用域

作用域链

函数内部也算一个作用域，当在预解释过程中，在函数里面没有找到a的变量值时，会从子作用域(fn1)跳到父级作用域查找，这种从里到外的查找的过程叫作用域链

一个参数就相当于一个局部变量

  • 当变量与函数重名,函数覆盖变量（因为变量值是undefined，

  • 函数与函数重名时，后面的的覆盖前面的

表达式 + - * / % Number() 参数，所有能改变变量值的操作都叫表达式，它是可以改变预解释的值的。

  2. 字符串
  3. 布尔值
  4. 函数
  5. 对象(element,[],{},null)

真

  1. 所有非0的数字(1,-1,1231,1.231)
  2. 所有的非空字符串(‘afbd’，’不可思异’,’ ‘有空格)
  3. true
  4. 所有的函数(function(){})

  1. 0、NaN
  2. 空字符串(‘’无空格)可以理解为空字符串就像在放屁一样，
    但里面包含了空格，就是想说但没说，所以会返回true

性能优化的方向

加载性能

一般最核心的就是减少请求数，因为每次请次数据都会产生3次TCP握手，如果每个资源都发出请求，那会很浪费时间。当合并完后要注意代码的压缩，减少体积

首屏加载时间 一般是指Dom ready时间

  • initial connection 网络等待时间
  • Waiting(TTFB)向服务器请求数据，服务器返回第一个数据包的时间，反应服务器性能
CDN

网络分发服务，主要是请求一些静态的资源.从最靠近的资源节点中获取数据

避免link中的空href和img中的空src

因为它也会发送请求的

使用Gzip压缩
把css放在顶部

减少重绘重排的几率

把js放在底部

避免js的阻塞

避免使用css表达式

主要是针对ie下的expression

把css和js抽取成外部文件

这样可以充分利用缓存机制，但并不代表内联就不好，内联的好处就是减少网络请求

get请求比post请求的速度要快
  • get请求常用于获取数据
  • post常用于修改数据
减少cookie的大小

因为在每次请求的时候都会带上cookie，如果cookie过大，会造成资源浪费

html prefetch

实现域名的预解释和页面的预加载
HTML5 prefetch-赖小赖小赖

$.getscript 动态加载script
对于非首屏的图片都使用lazyload

避免资源的浪费，同时避免了资源下载的抢占资源，比如说有100张图片，如果同时加载，那么通过有限的并发时，这100张图片就会占用网络带宽

js方面的

尽量不要使用with和eval

性能低下，而且有作用域转换问题，容易引起bug

高性能JavaScript DOM编程以及重排与重绘

event事件对象

  1. 当一个事件触发时，触发该事件的对象(鼠标或键盘)的相关信息会记录在event中

  2. 事件调用的函数，叫做事件函数，判断一个函数是不是事件函数，不是在它在声明的时候判断，而是看它是怎么被执行的

  3. 利用于飞机的黑匣子

  4. 例如鼠标的位置
  5. event对象一般都是在函数里面调用，并且这个函数必须是事件调用的，在函数外是没有内容的

    ie\chrome下,event是一个内置的全局对象(var event;)

    标准的浏览器下（ie8以上） 事件对象是通过函数的第一个参数传入的

  6. 一个函数是事件函数调用的，那个这个函数的第一个参数就是事件对象
  7. 兼容写法

clientX[Y]当鼠标点击发生时，鼠标到当前可视图的距离

  1. onmousemove当鼠标移动的时候,这是一个以时间间隔触发的函数，而不是每改变一次就触发一次，比如说，它是在50ms内监听一次，鼠标现在的位置与之前位置的值的变化。

    它是一个对象，代表浏览器的定位与导航

    • protocol 协议
    • hostname 主机名
    • port 端口号
    • host 主机地址
    • pathname 资源地址
    • search 查询字符串
    • hash 哈希部分

    5. window.location.replace(url)载入新的url，原页面不记录浏览历史

    dom Document object Model

    定义

    1. document:html页面
    2. 文档对象：页面中的元素
    3. 文档对象模型：是w3c为了让js能操作页面上的元素而创建出来的一套标准

    DOM节点类型

    1. Element 元素节点
    2. Text 文本节点(非标准类型的浏览器不支持,如ie8)

    方法

    1. Dom会把整个文档看成一棵树，页面上的所有元素会根据它们的层级来创建关系,例如父子节点，兄弟节点

    属性

    childNodes子节点列表集合

    1. 只读属性
    2. 它只包含到第一级子节点，不包含里面的节点
      • 标准下，包含了文本和属性节点，也包含非法嵌套的节点，如ul下面放直接放p
      • 非标准下，只包含元素节点，ie7以下不包含非法嵌套的节点,因为ie7下，会把这个不合理的p,放到li里面

    nodeType 当前元素的节点类型

    • 12种节点类型 w3c childNodes 属性
    1. 元素节点 1
    2. 属性节点 2
    3. 文本节点 3

    attributes获取元素的属性集合

    1. 只读的
    2. 因为一个元素的属性是有多个的，所以要加S
    1. 为元素节点添加样式

    children获取子节点列表集合

    1. 也是只读属性
    2. 只获取元素节点

    firstChild获取第一个子节点

    1. 只读
    2. 标准浏览器下，包含文本节点
    3. 非标准,只包含元素节点

    firstElementChild 获取第一个元素子节点

    lastChild获取最后一个节点

    lastElementChild获取最后一个标签元素

    nextSibling当前元素的下一个兄弟节点，nextElementSibling当前元素的下一个兄弟标签节点

    previousSibling当前元素的上一个兄弟节点,previousElementSibling当前元素的上一个兄弟标签节点

    parentNode 当前节点的父节点

    1. 只读

    offsetParent距离元素最近一个被positon定义过的父节点

    1. 如果没有定位父级标准浏览下，没定位父级，则为body，ie7下为html
      1.ie7下，如果当前元素的某个父级触发了layou,那么parentNode则指向这个元素

    offsetLeft,offsetTop获取当前元素距离父级定位元素的距离

    clientWidth,offsetWidth或 clientHeight,offsetHeight

    1. clientWidth[height] 可视区域宽度：样式宽度+padding
    2. offsetWidth[height] 占位宽：样式宽度+padding+border
    1. 获取元素到页面的距离

    元素的增、删、替换、移动

    document.createElement 创建元素

    appendChild 在父级下的内部后面增加元素

    insertBefore 在父级下，某个元素前插入元素

    1. 父级.insertBefore(插入内容，放在哪个元素前);

    replaceChild 在父级下，将某个元素剪切替换旧的元素

    1. 父节点.replaceChild(新节点，被替换的节点);

    removeChild 删除元素

    1. 被删除元素的父级.removeChild(要删除的元素);

    1.appendChild,insertBefore,replaceChild都可以对新增元素或者已存在元素进行操作，如果操作的元素是已存在的话，那个就相当于把已有元素剪切到新的地方

    性能优化

    因为在js中,DOM与es是属于两个不同的模块，就像两个不同的小岛，所以每次通过js来操作dom就相当于从一个岛到另外一个岛，两岛之间的桥每次通过都要收“手续费”(消耗性能)，所以我们要尽量减少过桥的次数（能够在独立完成的操作先独立完成）

    dom方法与innerHTML对比

    1. webkit 浏览器下，dom方法(如 createELement)效率比innerHTML高

    2. 其它浏览器下（中firefox），innerHTML的性能要高些

    DOM与浏览器的关系

    重排

    我们通过js进行DOM操作来改变元素的大小，位置时，让它的形状发生改变，就相当于改变了页面的内容，这样就消耗比较多的性能

    重绘

    当重排结束后，浏览器把重排完的元素显示出来，就产生是重绘，如果只改变背景颜色，那就只会触发重绘的过程，而不会重排

    尽量在插入元素前进行对元素内容的改变

    假如在元素插入后再进行内容的改变，这样就会让浏览器再次进入重排重绘的过程，造成资源的消耗
