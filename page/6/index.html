event事件对象

  1. 当一个事件触发时，触发该事件的对象(鼠标或键盘)的相关信息会记录在event中

  2. 事件调用的函数，叫做事件函数，判断一个函数是不是事件函数，不是在它在声明的时候判断，而是看它是怎么被执行的

  3. 利用于飞机的黑匣子

  4. 例如鼠标的位置
  5. event对象一般都是在函数里面调用，并且这个函数必须是事件调用的，在函数外是没有内容的

    ie\chrome下,event是一个内置的全局对象(var event;)

    标准的浏览器下（ie8以上） 事件对象是通过函数的第一个参数传入的

  6. 一个函数是事件函数调用的，那个这个函数的第一个参数就是事件对象
  7. 兼容写法

clientX[Y]当鼠标点击发生时，鼠标到当前可视图的距离

  1. onmousemove当鼠标移动的时候,这是一个以时间间隔触发的函数，而不是每改变一次就触发一次，比如说，它是在50ms内监听一次，鼠标现在的位置与之前位置的值的变化。

onclick对应的事件函数)而中断。

    BOMbrowser object model 浏览器对象模型

    属性

    属性名 描述
    navigator 浏览器信息
    location 浏览器定位和导航
    history 窗口浏览历史
    screen 屏幕信息

    window.navigation浏览器信息

    1. window.navigation.platform当前运行系统信息
    2. window.navigator.userAgent浏览器信息,可以用来判断当前用户所运行的是什么浏览器

    window.location浏览器链接

    它是一个对象，代表浏览器的定位与导航

    • protocol 协议
    • hostname 主机名
    • port 端口号
    • host 主机地址
    • pathname 资源地址
    • search 查询字符串
    • hash 哈希部分
    1. window.location.href返回完整的页面路径，以通过修改它来完成页面跳转
    2. window.location.searchurl地址?后的内容
    3. window.location.hashurl地址#后的内容
    4. window.location.assign(url)载入新的url,原页面记录浏览历史
    5. window.location.replace(url)载入新的url，原页面不记录浏览历史
    6. window.location.reload() 重新载入页面

    history浏览历史

    1. history.back() 传入正整数，表示要后退的步长
    2. history.foward() 传入正整数，表示要前进的步长
    3. history.go() 传入整数，表示跳的步长

    screen屏幕信息

    • width 屏幕宽度
    • height 屏幕高度
    • availHeight 浏览器可用高度（屏幕高-系统底栏高度）
    • availWidth 浏览器可用宽度

    窗口尺寸

    可视区尺寸

    1. 宽 document.documentElement.clientWidth
    2. 高 document.documentElement.clientHeight

    滚动距离

    1. document.documentElement.scrollTop[scrollLeft] 非chrome下
    2. document.body.scrollTop[scrollLeft] chrome因为浏览器的滚动条距离应该是属于body下的

    内容尺寸

    1. scrollHeight[scrollWidth]元素实际占据的内容的大小，其实就是除了边框以外，内里的内容从顶部到底部的距离

    文档高度

    1. document.body.offsetHeight 获取整个文档内容的高度

    window下的方法

    方法名 描述
    alert(),comfirm(),prompt() 对话框
    setTimeout(),setInterval() 定时器
    open(),close() 打开关闭窗口

    BOM下的大多数方法都是在window下的，而js下规定，如果方法是window下的，可以不写如window.open()可以写成open()

    对话框，页面会阻塞浏览器后面线程执行

    alert() 警告框

    comfirm() 确定框

    点“确定”，返回的是true,点“取消”返回的是false

    prompt() 输入框

    用户点击确定后，会返回用户在输入框输入的内容

    open(网页url,打开的方式target)打开一个新窗口

    1. 如果url为空，默认为空
    2. 如果target为空，默认为新窗口打开
    3. 它会返回新窗口的window对象

    close()关闭指向的window

    • ff:无法关闭当前窗口
    • chrome:关闭当前窗口
    • ie:让用户确定是否关闭
    • 可以关闭通过js打开的新窗口

    事件

    事件名 出发时机
    load 文档和所有图片加载完成时
    unload 当离开当前文档时
    beforeunload 与unload类似，但它提供询问用户是否确认离开的机会
    resize 浏览器窗口大小发生变化时
    scroll 拖动滚动条时

    onload 文档和所以图片加载完成时会触发

    onscroll当滚动条滚动时

    onresize当窗口大小被调整时

    这两个都是按时间来算的，比如说，它们都是在5毫秒里检测一次，但在5毫秒里拖动了多次，那依然只是会出发一次

    dom Document object Model

    定义

    1. document:html页面
    2. 文档对象：页面中的元素
    3. 文档对象模型：是w3c为了让js能操作页面上的元素而创建出来的一套标准

    DOM节点类型

    1. Element 元素节点
    2. Text 文本节点(非标准类型的浏览器不支持,如ie8)

    方法

    1. Dom会把整个文档看成一棵树，页面上的所有元素会根据它们的层级来创建关系,例如父子节点，兄弟节点

    属性

    childNodes子节点列表集合

    1. 只读属性
    2. 它只包含到第一级子节点，不包含里面的节点
      • 标准下，包含了文本和属性节点，也包含非法嵌套的节点，如ul下面放直接放p
      • 非标准下，只包含元素节点，ie7以下不包含非法嵌套的节点,因为ie7下，会把这个不合理的p,放到li里面

    nodeType 当前元素的节点类型

    • 12种节点类型 w3c childNodes 属性
    1. 元素节点 1
    2. 属性节点 2
    3. 文本节点 3

    attributes获取元素的属性集合

    1. 只读的
    2. 因为一个元素的属性是有多个的，所以要加S
    1. 为元素节点添加样式

    children获取子节点列表集合

    1. 也是只读属性
    2. 只获取元素节点

    firstChild获取第一个子节点

    1. 只读
    2. 标准浏览器下，包含文本节点
    3. 非标准,只包含元素节点

    firstElementChild 获取第一个元素子节点

    lastChild获取最后一个节点

    lastElementChild获取最后一个标签元素

    nextSibling当前元素的下一个兄弟节点，nextElementSibling当前元素的下一个兄弟标签节点

    previousSibling当前元素的上一个兄弟节点,previousElementSibling当前元素的上一个兄弟标签节点

    parentNode 当前节点的父节点

    1. 只读

    offsetParent距离元素最近一个被positon定义过的父节点

    1. 如果没有定位父级标准浏览下，没定位父级，则为body，ie7下为html
      1.ie7下，如果当前元素的某个父级触发了layou,那么parentNode则指向这个元素

    offsetLeft,offsetTop获取当前元素距离父级定位元素的距离

    clientWidth,offsetWidth或 clientHeight,offsetHeight

    1. clientWidth[height] 可视区域宽度：样式宽度+padding
    2. offsetWidth[height] 占位宽：样式宽度+padding+border
    1. 获取元素到页面的距离

    元素的增、删、替换、移动

    document.createElement 创建元素

    appendChild 在父级下的内部后面增加元素

    insertBefore 在父级下，某个元素前插入元素

    1. 父级.insertBefore(插入内容，放在哪个元素前);

    replaceChild 在父级下，将某个元素剪切替换旧的元素

    1. 父节点.replaceChild(新节点，被替换的节点);

    removeChild 删除元素

    1. 被删除元素的父级.removeChild(要删除的元素);

    1.appendChild,insertBefore,replaceChild都可以对新增元素或者已存在元素进行操作，如果操作的元素是已存在的话，那个就相当于把已有元素剪切到新的地方

    性能优化

    因为在js中,DOM与es是属于两个不同的模块，就像两个不同的小岛，所以每次通过js来操作dom就相当于从一个岛到另外一个岛，两岛之间的桥每次通过都要收“手续费”(消耗性能)，所以我们要尽量减少过桥的次数（能够在独立完成的操作先独立完成）

    dom方法与innerHTML对比

    1. webkit 浏览器下，dom方法(如 createELement)效率比innerHTML高

    2. 其它浏览器下（中firefox），innerHTML的性能要高些

    1. 只获取元素节点 children firstElementChild
    2. 同时获取元素节点和内部的文本节点 childNodes firstChild

    DOM与浏览器的关系

    重排

    我们通过js进行DOM操作来改变元素的大小，位置时，让它的形状发生改变，就相当于改变了页面的内容，这样就消耗比较多的性能

    重绘

    当重排结束后，浏览器把重排完的元素显示出来，就产生是重绘，如果只改变背景颜色，那就只会触发重绘的过程，而不会重排

    尽量在插入元素前进行对元素内容的改变

    假如在元素插入后再进行内容的改变，这样就会让浏览器再次进入重排重绘的过程，造成资源的消耗

    ajax

    发表于 2015-12-12 |

    ajax（Asynchronous Javascript And XML,异步的javascript和xml），无刷新的情况下进行跟后台数据交互的，可应用于注册表单验证，在线聊天等场景。

    1. 通过标准下的new XMLHttpRequest，ie下的new ActiveXObject()来实例XHR对象
    2. 通过open(‘请求方法’.’请求地址’,是否异步请求)来建议一个数据请求内容，然后通过send方法来与服务器建立链接，假如是POST请求，则可以打请求的内容写到require.send(请求内容)中去。
    3. 当发送请求后，我们可以通过，require.onreadystates为监听请求进度（0.初始化，1.open方法民被调用，2.接收请求头信息，3处理中响应主体，4处理完成）及require.status(200请求成功)
    4. 当这些都成功后，我们通过服务器返回的responseText，利用js的DOM操作动态更新数据
    5. 假如遇到跨域问题（协议，子域名，主域名，端口号不同），需要不同域名之间访问数据，可以通过服务器代理，get情况下考虑使用JSONP（script标签可以请求数据），无需兼容ie10以下版本的XHR2方法来解决跨域。
