ajax

发表于 2015-12-12 |

ajax（Asynchronous Javascript And XML,异步的javascript和xml），无刷新的情况下进行跟后台数据交互的，可应用于注册表单验证，在线聊天等场景。

  1. 通过标准下的new XMLHttpRequest，ie下的new ActiveXObject()来实例XHR对象
  2. 通过open(‘请求方法’.’请求地址’,是否异步请求)来建议一个数据请求内容，然后通过send方法来与服务器建立链接，假如是POST请求，则可以打请求的内容写到require.send(请求内容)中去。
  3. 当发送请求后，我们可以通过，require.onreadystates为监听请求进度（0.初始化，1.open方法民被调用，2.接收请求头信息，3处理中响应主体，4处理完成）及require.status(200请求成功)
  4. 当这些都成功后，我们通过服务器返回的responseText，利用js的DOM操作动态更新数据
  5. 假如遇到跨域问题（协议，子域名，主域名，端口号不同），需要不同域名之间访问数据，可以通过服务器代理，get情况下考虑使用JSONP（script标签可以请求数据），无需兼容ie10以下版本的XHR2方法来解决跨域。

同步与异步

  • 同步是做完一件事再做另外一件事
  • 异步是同时做多件事，服务器操作与本地操作之间不同互相阻塞

一个完整的HTTP请求过程，通常有7个步骤：

  1. 建立TCP链接；
  2. Web浏览器向Web服务器发送请求命令；
  3. Web浏览器发送请求头信息
  4. Web服务器应答；
  5. Web服务器发送应答头信息
  6. Web服务器向浏览器发送数据
  7. Web服务器关闭TCP链接
  • http是计算机通过网络进行通信的规则
  • 它无状态的服务协议，不建立持久链接，不会保存之前的数据

一个HTTP请求一般由四部分组成

（get：http:www.baidu.com/?a.html/username=weison）

  1. HTTP请求的方法或动作，比如是GET还是POST请求
    • get 一般用于信息获取（查询），使用URL传递参数，获取的信息对任何人可见，对所发送信息的数量也有限制，一般在2000个字符。（GET方法是幂等的,无论查询多少次，获取的信息都是一样的）
    • post 一般用于修改服务器上的资源，对所发信息的数量无限制。
  2. 正在请求的URL，总得知道请求的地址是什么吧
  3. 请求头，包含一些客户端环境信息，身份验证信息等
  4. 请求体，也就是请求正文，请求正文中可以包含客户提交的查询字符串信息，表单信息等等

HTTP响应，由三部分组成

  1. 一个数字和文字组成的状态码;
  • 信息类(1xx)，表示收到Web浏览器请求，正在进一步的处理中
  • 成功(2xx)，表示用户请求呗正确接收，理解和处理。例如：200 OK
  • 重定向(3xx)，表示请求没有成功，客户必须采取进一步的动作
  • 客户端错误(4xx)，表示客户端提交的请求有错误，例如：404 Not Found，意味着请求中所引用的文档不存在
  • 服务器错误(5xx)，表示服务器不能完成对请求的处理。例如500
  1. 响应头（服务器信息），响应头也和请求头一样包含许多有用的信息，例如服务器类型、日期时间、内容类型和长度等；
  2. 响应体，也就是响应正文，根据用户的请求内容返回的响应数据。

XMLHttpRequest 实例化

  • 标准浏览器下,new XMLHttpRequest();
  • ie6及以下,new ActiveXObject()

发送请求XMLHttpRequre

open(method,url,async) 调用http请求

method 代表是发送请求的方式，是使用get还是post,不区分大小写，一般用大写

url 请求地址，可以是相对地址或绝对地址

async 请求是同步还是异步的，ajax一般同步true值，而采用同步就把这个值设为false,此值默认就为true，所以一般不用填写

send(string) 把请求发送到服务器

  • 使用get方法是没有主体的，所以会把所有参数都拼在url里面，所以可以不填写或者写为sent(none)

获取服务器响应数据 XMLHttpRequest

responseText 获取字符串形式的响应数据

responseXML 获取xml形式的响应数据（现在比较小用，较常用的是json）

status和statusText 以数字和文本形式返回HTTP状态码

getAllResponeHeader() 获取所有响应的报头

getResponeseHeader() 响应中某个字段的值

readyStatus 获取服务器的响应变化

  • 0：请求初始化，open还没有调用
  • 1：服务器连接已建立，open已经调用了
  • 2：请求已接收，也就是接收到头信息了
  • 3：请求处理中，也就是接收到响应主体了
  • 4：请求已完成，且响应已就绪，也就是响应完成了

onreadystatuschange来监听服务器的服务器的响应变化

JSON

  1. javascript对象表示法(javascript object Notation)
    • 采用键值对的方式来组织
    • 独立于语言，不管什么语言，都可以解释JSON
  2. JSON的长度比XML的格式短小
    • 读写速度快
    • 可使用js内建方法直接解释，转换成js对象
  3. Json的键名需要加上“”，如 “name”:”weison”;
  4. Json模板

Json的解释

eval
JSON.parse

eval不会校验数据是否合法，而且如果json里面写了命名，它也会执行，这是不安全的，所以为了安全起建，还是使用JSON.parse比较好

跨域

定义


不同的域名不间请求数据，就叫跨域
因为js不允许跨域来操作DOM（同源策略），所以a.html的js文件不能操作b.html下的DOM

解决跨域问题方案

jsonp
  • json+padding
  • 可以解决主流浏览器下的get请求跨域问题
  • 它是把script的src标签当成的请求的地址，它可以接受任意类型的文件（如baidu.com/php?name=weison&&age=20），只要最后返回的数据是json或者是js，它就不会有问题。
CORS跨域资源共享\XHR2

Cross-origin resource sharing,利用html5下的XMLHttpRequest level2 实现跨域（ie10以下不支持）。当页面发起异步的AJAX请求时，浏览器会带上origin头，然后由后台的Access-Control-Allow-Origin来决定是否允许请求数据,如果允许，则浏览器可以拿到响应数据，否则请求会被驳回。

www.b.weison.org 内设置 document.domain = "weison.org"
