利用git进行版本控制

发表于 2017-02-11 |

git作用

  • 可以回溯到之前的任意版本
  • 解决多人协作时的冲突问题

github

  • geek平台，可以找到开源项目以及作远程仓库之用
  • 可视化工具github desktop

svn与git的区别

  • svn采用的是集中式的管理方式,所有代码都存放在中
  • git采用的是分布式的管理方式

使用git

  • 复制仓库 git clone [url]
  • 设置贡献者,用于在多人协作时识别coder身份
  1. git config –global user.name “名字”
  2. git config –global user.email “邮箱地址”
  3. 查看开发者名称 git config –global user.name
  4. 查看开发者邮箱 git config –global user.email
  5. 查看所有git的项目配置文件 git config –list
  • 分支
  1. 为了保证新增功能不影响主项目的开发，而创建一个与主分支一样的副本文件，当这个副本开发相对完善后再合并到主分支，形成一个新的稳定版本
  2. 默认分支 master
  • git的三个区
  1. 工作区，正在编写代码的开发环境
  2. 暂存区
    • 作为工作区与版本库的桥梁，减少误操作的风险
    • 当工作区的文件在误操作时可以通过暂存区复原，也保证未编写完整的代码提示到版本库上
    • 分支处理，当要临时切换到另外的分支进行开发工作时，可以把正在编写的文件提交到暂存区，然后再切换到另一个分支，当有需要时再切换回来，从暂存区取出代码
  3. 版本库
  • git 命令
  1. git status 查看工作区和暂存区的状态，是否有经过编辑
    • Changes not staged for commit 说明已跟踪文件的内容发生了变化，但还没有放到暂存区
  2. git add 文件名 => git add index.html 将工作区文件添加到暂存区
  3. git add . 将工作区所有文件添加到暂存区
  4. git commit -m “修改的内容” => git commit -m “change on file” 将文件从暂存区添加到版本库
  5. git commit -a -m “修改内容” => git commit -a -m “hello” 将文件同步保存到暂存区并添加到版本库,-a就是简写的add
  6. git log 查看提示记录,当提交记录过多时，不会一下全部打印出来，不过可以按回车来查看，然后按“q”键即可退出
  7. git diff 对比命令
  8. git diff 直接使用对比工作区与暂存区的内容
  9. git diff –cached 或 git diff –staged 暂存区与版本库内容对比
  10. git diff master 对比本地与版本库
    1.撤消
  11. git reset HEAD 文件名 => git reset HEAD drag.js 将你出去到暂存区的drag.js文件取回到暂存区，相当于取消提交
  12. git checkout – 文件名 => git checkout – drag.js 将文件回复到版本库上的内容
  13. git commit –amend => git commit -a -m “修正上一版本错误” –amend 覆盖上一次提交到版本库的文件，再重新上传一次
    1.删除
  14. git rm 文件名 => git rm test.txt ，当工作区的文件删除后，使用git rm test.txt 删除暂存区的文件
  15. git rm -f 文件名 => git rm -f test.txt 同时删除本地及暂存区的文件
  16. git rm –cached => git rm -cached test.txt 只删除暂存区的文件
    1.恢复
  17. git checkout commitId 文件名 => git checkout 50e7b8e0610 drag.js 使用git log 查看提交的id,然后恢复具体的文件
  18. git reset –hard commitID => git reset –hard 50e7b8e061 恢复到某个特定版本
    • git reset –hard head^ 返回上一个版本
    • git reset –hard head~3 返回前3个版本
  19. git reflog 查看所有操作，当切换到旧版本后git log没有了新的记录，可以通过git reflog查看所有操作，然后然后git reset –hard commit 来回来新版本
    1.提交代码
  20. git remote 远端名称
  21. git remote -v 远程地址
  22. git push 名字 分支名
  23. haha

创建并切换到该分支 git checkout -b [分支名称]
合并分支 git merge [分支名称]
删除分支 git branch -d [分支名称]

安装

  1. mysysgit

  2. beyondcompare 文档比较软件

    生成钥匙

  3. ssh-keygen -t rsa -c "email地址"
  4. cat ~/.ssh/id_rsa.pub
  5. 把它粘贴到github的ssh key上

创建仓库

  1. touch readme.md创建描述文件
  2. git init 初始化本地目录
  3. git add [fileName]让本地文件提交到暂存区
  4. git commit -am '[更新描述]' -am的意思，-a代表修改的文件,m代表删除的文件,所以这句话的意思是把所有添加、修改、删除都提交
  5. git remote add [远程地址名称] [远程地址路径]如git remote add origin abc.git就相当于把abc.git起了个名字叫origin,下次提交的时候写git push origin master就相当于推送内容到远端origin下的master分支
  6. git pull更新同步一下远端的代码，

分支操作

  1. git branch -a查看当前本地的所有分支，红色的不加星号的就是远程分支
  2. git branch dev创建一个dev分支
  3. git checkout dev切换到dev分支
  4. git push -u origin dev把本地的分支推送到远程

代码合并

  1. git add .把最新的修改添加到更新列
  2. git commit -am [更新描述]
  3. git pull [远端地址] [分支名称]
  4. git push [远端地址] [分支名称]把内容更新到远端
  5. git merge [分支名称]
  6. git merge master把主干master上的所有代码合并到当前分支
  7. 假如冲突(CONFLCT)，要先解决冲突

代码回溯

  1. git log -l 3查看最近3次的提交
  2. 找到SHA码
  3. git reset --hard [SHA码]将本地的代码强制回到了[SHA]对应的文件(但复原后直接拒绝，会被拒绝),可以使用
    git reset -f --hard [SHA码]来强行提交，不过这样会把在这个版本后的记录都会清除（不建议使用）,为了避免
    这种情况，可以使用git revert [SHA码]来返回指定的版本，github会把这次回滚理解成一次新的修改，这样会保留记录安全了很多。
